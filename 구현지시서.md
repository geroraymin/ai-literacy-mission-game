# AI 리터러시 미션 게임 — 구현지시서

> 이 문서가 게임 구현의 단일 기준이다. 구현 AI(Codex/Claude)는 이 문서 전체를 요구사항으로 삼는다.
> 작업 위치: 이 프로젝트의 `app\` 폴더 (Vite + React + TypeScript 스캐폴드 준비됨)

# 역할
너는 교육용 게임 기획자이자 시니어 웹게임 개발자다. 청소년 교육 콘텐츠 작성 경험과
2D 웹게임(타일맵, 게임 루프) 구현 경험을 모두 갖추고 있다.

# 목표
React + TypeScript + Vite로 "AI 연구소 탐험" 2D 탑다운 미션 게임 MVP를 만든다.
- 대상: 중학생 (모든 텍스트는 중학생이 이해할 수 있는 쉬운 한국어)
- 플레이 분량: 20~30분, 미션 5개
- 용도: 학교 수업 1차시(45분)에서 바로 활용 가능한 완성품
- 교육 주제: AI 윤리, 데이터 편향, 프롬프트 작성, 개인정보 보호, 생성형 AI 활용

# 게임 형태
- 2D 탑다운(위에서 내려다보는) 타일맵. 타일 16px 기준, 정수 배율(x2~x3)로 확대 렌더링
- 맵: AI 연구소 1개 층. 중앙 로비 + 미션 방 5개(데이터실, 프롬프트 랩, 보안실,
  생성 AI 스튜디오, AI 윤리위원회). 방 하나 = 미션 하나
- 렌더링은 Canvas 사용, UI(대화창/퀴즈/HUD)는 React 컴포넌트로 분리

# 조작 (PC + 태블릿 모두 지원)
- PC: 방향키/WASD 이동, Space 또는 Enter로 대화·상호작용
- 태블릿: 화면 좌하단 가상 방향패드 + 우하단 상호작용 버튼 (터치 이벤트, 포인터 이벤트 처리)
- 터치 기기 여부를 감지해서 가상패드를 자동 표시/숨김
- 가로 태블릿(1024px~)과 PC 해상도 모두에서 레이아웃이 깨지지 않을 것

# 필수 기능 (각 항목은 완성 기준까지 충족해야 함)
1. 캐릭터 이동 — 4방향 이동, 벽·가구 충돌 처리, 걷기 애니메이션(2프레임 이상)
2. 맵 탐험 — 문으로 방 간 이동, 현재 방 이름 표시
3. NPC 대화 — 대화창(이름 + 초상 + 텍스트), Space/터치로 다음 문장, 순차 대화
4. 퀘스트 시스템 — 수락 → 진행 중 → 완료 상태 관리, 현재 목표를 HUD에 항상 표시
5. AI 개념 퀴즈 — 미션당 3~4문항. 4지선다, OX, "잘못된 것 찾기" 유형을 섞을 것
6. 정답/오답 피드백 — 정답: 개념 한 줄 정리 / 오답: 왜 틀렸는지 설명 후 재도전 허용
   (재도전 시 획득 점수 감소)
7. 점수 + 배지 — 미션별 배지 5개, 총점. 배지는 엔딩에서 한눈에 보이게
8. 미션 진행률 — 상단 HUD에 "미션 n/5" + 배지 아이콘
9. 엔딩 화면 — 배지 요약, 배운 개념 복습 카드 5장, 점수에 따른 칭호
10. 리스타트 — "처음부터" 버튼 + localStorage 저장으로 "이어하기" 지원

# 미션 내용 (방 하나씩 배정)
1. 보안실 — 개인정보 보호: 개인정보(이름, 주소, 전화번호 등)가 포함된 프롬프트 찾아내기
2. 프롬프트 랩 — 프롬프트 작성: 좋은 프롬프트와 나쁜 프롬프트 구분하고 개선안 고르기
3. 데이터실 — 데이터 편향: 편향된 학습 데이터 사례 고르기 (예: 특정 성별/지역만 있는 데이터)
4. 생성 AI 스튜디오 — 결과 검증: AI가 지어낸 답(환각) 판별, 올바른 검증 방법 선택
5. AI 윤리위원회 — AI 윤리: 생성형 AI 활용 시 지켜야 할 것 (표절 금지, 출처 밝히기)

- 퀴즈 문항·NPC 대사는 코드에 섞지 말고 `src/data/` 폴더에 별도 데이터 파일로 분리할 것
  (교사가 코드를 몰라도 문항을 고칠 수 있어야 한다)
- 각 미션은 "상황 제시(NPC 대화) → 퀴즈 → 피드백 → 개념 정리" 흐름을 따를 것

# 그래픽 (중요 — 반드시 이 순서로 시도)
**1순위: Kenney CC0 무료 에셋 사용**
- 프로젝트 루트의 `assets\` 폴더에 이미 준비된 Kenney 에셋이 있으면 그것을 사용한다
- 없으면 https://kenney.nl 의 탑다운 타일셋(예: Tiny Town, Roguelike/RPG pack 계열)을
  다운로드해서 `app/public/assets/`에 배치하고 스프라이트시트로 사용
- 사용한 팩 이름과 출처를 README와 `SOURCES.md`에 기록

**2순위 (에셋 다운로드가 불가능한 환경일 때만): 코드 내 SVG/Canvas 스프라이트 직접 제작**

**어느 쪽이든 아래 아트 디렉션을 반드시 지킬 것:**
- 고정 팔레트 8~12색만 사용하고, 팔레트를 상수 파일에 정의할 것
- 화면에 이모지를 나열하는 방식 금지 (이모지는 배지·HUD 아이콘 정도만 허용)
- 캔버스에 image-rendering: pixelated 적용, 타일은 정수 배율로만 확대
- UI 폰트: Galmuri 같은 한글 픽셀 웹폰트. 대화창·버튼은 레트로 RPG 스타일로 통일
- 전체적으로 밝고 친근한 톤이되, 색과 스타일이 화면마다 따로 놀지 않을 것

# 코드 구조
```
app/src/
  components/  — React UI (대화창, 퀴즈 모달, HUD, 가상 방향패드, 엔딩 화면)
  game/        — 게임 루프, Canvas 렌더링, 입력 처리, 충돌 판정
  systems/     — 퀘스트 상태, 퀴즈 채점, 배지/점수, localStorage 저장
  data/        — 맵 데이터, NPC 대사, 퀴즈 문항, 팔레트 상수
```
- TypeScript strict 모드, any 금지
- 파일 하나당 500줄 이하로 분리
