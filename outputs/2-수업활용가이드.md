# AI 연구소 탐험대 — 수업 활용 가이드 (교사용)

- 대상: 중학생 (초6~고1까지 무리 없음)
- 소요: 1차시 45분 (게임 플레이 약 20~30분)
- 준비물: 학생용 PC 또는 태블릿(크롬 브라우저), 게임 실행 주소
- 다루는 개념: 개인정보 보호 · 프롬프트 작성 · 데이터 편향 · AI 결과 검증(환각) · AI 윤리

## 실행 방법

```bash
# app 폴더에서 (Node.js 설치 필요)
npm install     # 처음 한 번만
npm run dev     # 실행 → http://localhost:5173 접속
```

- 여러 학생이 접속하려면 `npm run dev -- --host`로 실행하고, 같은 와이파이에서 교사 PC의 IP 주소로 접속합니다.
- 인터넷 없이도 동작합니다(로컬 실행). 학생 계정·로그인이 필요 없습니다.
- 진행 상황은 각자 브라우저에 저장되므로, 시간이 모자라면 다음 시간에 "이어하기"로 계속할 수 있습니다.

## 45분 차시안

| 시간 | 단계 | 내용 |
|---|---|---|
| 5분 | 도입 | "AI에게 아무거나 다 말해도 될까?" 질문으로 시작 → 게임 스토리 소개 (AI 비서 '아리'가 잘못된 사용법 때문에 고장) |
| 25분 | 플레이 | 개별 또는 2인 1조 플레이. 미션 순서는 자유. 교사는 순회하며 막힌 학생의 오답 해설을 함께 읽어줌 |
| 10분 | 토론 | 아래 토론 질문 중 2~3개 선택. "가장 어려웠던 문제"부터 시작하면 자연스러움 |
| 5분 | 정리 | 엔딩 화면의 "오늘 배운 다섯 가지" 복습 카드를 다 같이 읽고, 실천 다짐 한 가지씩 발표 |

> 팁: 점수(400점 만점)는 경쟁보다 "재도전해도 배지는 똑같이 받는다"는 점을 강조해 주세요. 틀린 학생이 오답 해설을 더 많이 읽게 되는 구조입니다.

## 미션별 토론 질문 (각 2개)

**🔒 보안실 — 개인정보 보호**
1. AI에게 절대 보내면 안 되는 정보를 3가지 이상 말해볼까? 왜 위험할까?
2. 친구가 채팅 내용을 통째로 AI에 붙여넣으려고 한다면 뭐라고 말해줄까? (친구의 개인정보도 '남의 개인정보'라는 점)

**✏️ 프롬프트 랩 — 프롬프트 작성**
1. "숙제 도와줘"를 역할·대상·형식이 들어간 좋은 프롬프트로 직접 바꿔보자.
2. 좋은 프롬프트를 쓰는 능력이 왜 "질문하는 능력"과 같다고 할까?

**⚖️ 데이터실 — 데이터 편향**
1. 우리 주변에서 편향된 데이터로 만들어진 AI가 있다면 어떤 문제가 생길까? (예: 얼굴 인식, 채용 AI)
2. "AI가 그렇게 판단했으니 공정하다"는 말의 문제점은 무엇일까?

**🔍 생성 AI 스튜디오 — 결과 검증**
1. AI의 답을 꼭 교차 확인해야 하는 경우와 그냥 써도 되는 경우를 나눠보자.
2. AI가 지어낸 가짜 정보(환각)에 속았던 경험이나 뉴스 사례가 있을까?

**🌱 AI 윤리위원회 — AI 윤리**
1. AI로 만든 그림·글에 "AI 사용"을 밝히는 것이 왜 나를 지키는 일이기도 할까?
2. 딥페이크가 장난으로 시작해도 큰 피해가 되는 이유는 무엇일까?

## 평가 활용 (선택)

- 엔딩 화면을 캡처해 제출하게 하면 수행 확인이 됩니다 (칭호 + 미션별 점수 표시)
- 토론 후 "오늘 배운 다섯 가지 중 내가 실천할 한 가지"를 한 줄로 쓰게 하면 정리 활동이 됩니다

## 퀴즈 문항 수정 방법 (코딩 몰라도 가능)

문항·대사는 코드와 분리되어 있어 문장만 고치면 됩니다.

1. `app\src\data\quizzes.ts` 파일을 메모장/VS Code로 연다
2. 고치고 싶은 문장(따옴표 안)을 수정한다
   - `prompt:` 문제 / `text:` 보기 / `explain:` 고른 뒤 보여줄 설명
   - `correct: true`가 붙은 보기가 정답 (문항마다 1개만!)
3. 문항을 추가하려면 기존 문항 블록( `{ id: ... },` )을 복사해 붙여넣고 내용을 바꾼다
4. 저장하면 개발 서버가 자동으로 새로고침된다

NPC 대사는 `app\src\data\dialogues.ts`에서 같은 방법으로 수정합니다.
