1게임 소개 · 학습 목표

플레이어는 AI 연구소의 신입 탐험 대원이 되어, 잘못된 사용법 때문에 고장 난 AI 비서 "아리"를 구하기 위해 다섯 개의 방을 탐험합니다. 방마다 연구원 NPC가 내는 퀴즈 미션(각 4문항)을 해결하면 배지를 얻고, 5개를 모으면 엔딩에서 칭호와 복습 카드를 받습니다.

💡 팁주소가 길면 QR코드로 만들어 칠판에 붙이거나, 단축주소 서비스를 이용하세요.

방법 B. 교사 PC를 서버로 — 인터넷이 안 되는 교실용

교사 PC와 학생 태블릿이 같은 와이파이에 연결되어 있으면, 인터넷 없이도 가능합니다. (교사 PC에 Node.js 설치 필요)

  1. 교사 PC에서 프로젝트의 app 폴더로 이동해 터미널을 연다
  2. npm install 실행 (처음 한 번만)
  3. npm run dev -- --host 실행
  4. 화면에 나오는 Network 주소 (예: http://192.168.0.15:5173)를 학생들에게 알려준다
⚠️ 주의학교 와이파이가 기기 간 통신을 차단(AP 격리)하는 경우가 있습니다. 수업 전에 태블릿 1대로 미리 접속 테스트를 해보세요.

💡 제일 쉬운 방법Claude Code에게 시키세요 — "보안실 2번 문제를 이렇게 바꿔줘", "플레이어 캐릭터를 기사로 바꿔줘"라고 하면 수정하고 확인해서 커밋 제안까지 해줍니다.

7수정 → 인터넷 반영 절차

수정 자체는 인터넷이 필요 없습니다. 인터넷 주소(방법 A)에 반영할 때만 GitHub에 올리면 됩니다.

  1. 파일 수정 (6번 가이드 참고)
  2. app 폴더에서 npm run dev → localhost:5173에서 직접 확인
  3. 프로젝트 폴더에서 터미널을 열고:
    git add -A → git commit -m "fix: 보안실 2번 문항 수정" → git push
  4. 1~2분 기다리면 인터넷 주소에 자동 반영 (GitHub Actions가 자동으로 다시 빌드·배포)
ℹ️ 배포가 됐는지 확인하려면GitHub 저장소 → Actions 탭에서 초록색 체크(✓)가 뜨면 완료. 게임 페이지에서 Ctrl+F5(강력 새로고침)로 확인하세요.
💡 Claude Code를 쓴다면수정 후 "커밋하고 올려줘" 한 마디면 2~4번이 자동으로 처리됩니다.

8문제 해결 (FAQ)

증상해결
화면이 하얗게/까맣게만 나온다새로고침(F5) → 안 되면 Ctrl+F5. 직접 수정한 직후라면 마지막 수정 파일의 따옴표·쉼표 확인
npm run dev가 "port 5173 already in use"이미 켜져 있는 것 — 기존 터미널을 닫거나, 그냥 열려 있는 주소를 쓰면 됨
npm 명령을 찾을 수 없다고 나온다Node.js 미설치 — nodejs.org에서 LTS 버전 설치 후 터미널 다시 열기
태블릿에서 교사 PC 주소로 접속이 안 된다 (방법 B)① 같은 와이파이인지 확인 ② -- --host 옵션을 붙였는지 확인 ③ Windows 방화벽 허용 팝업에서 "허용" ④ 학교망 AP 격리면 방법 A 사용
학생이 "이어하기"가 안 보인다고 한다저장은 기기+브라우저 단위 — 지난 시간과 같은 기기·같은 브라우저인지 확인. 시크릿 모드는 저장 안 됨
진행을 처음부터 다시 시키고 싶다타이틀에서 "새로 시작"을 누르면 기존 저장이 지워지고 처음부터 시작됨
이모지 배지가 네모(□)로 보인다아주 오래된 PC의 이모지 폰트 문제 — 게임 진행에는 지장 없음. 크롬 최신 버전 권장
push 후에도 인터넷 주소에 반영이 안 된다GitHub 저장소 Actions 탭에서 실패(✗) 여부 확인 → 실패 로그를 Claude Code에 보여주면 해결 가능
